## یک نگاه ۳۰ ثانیه‌ای

```text
جذب  →  مشاوره اولیه  →  پیگیری  →  مشاوره تخصصی  →  تشکیل پرونده
                                                      ↓
                         موفق ✓          پذیرش → ویزا → اعزام
                         مختومه ✗ (عدم علاقه / …)
```

| الان | بعد از پنل |
|------|------------|
| ارزیابی و مشاوره جدا از هم | یک **پرونده** برای هر مشتری |

| وقت مشاوره فقط در ادمین خشک | **تقویم شمسی** برای هر اپراتور |
| جستجوی سخت | سرچ هوشمند نام / موبایل / کد پرونده |
| موبایل برای کار روزمره مناسب نیست | UI **ریسپانسیو**، انگشتی، RTL |

## روز کاری اپراتور (سناریوی واقعی)

### صبح — باز کردن پنل
1. لاگین با یوزر خودش  
2. داشبورد نشان می‌دهد: **۱۲ پیگیری امروز** · **۵ عقب‌افتاده** · **۳ جلسه** · **۸ لید جدید**  
3. اول عقب‌افتاده‌ها، بعد امروز

### تماس ۲ دقیقه‌ای
1. روی کارت مشتری → «ثبت تماس»  
2. نتیجه: پاسخ داد / عدم پاسخ / واتساپ / مشغول  
3. یک خط یادداشت  
4. موعد بعدی: امروز شب / فردا / ۳ روز / تاریخ شمسی  
5. ذخیره — برمی‌گردد به صف

### وقتی مشتری جدی است
- مرحله را به «مشاوره تخصصی» ببرد  
- از همان پرونده وقت بگذارد → در تقویم شمسی می‌نشیند  
- یا «تشکیل پرونده» بزند و چک‌لیست اعزام شروع شود

### وقتی مشتری نمی‌خواهد
- دکمهٔ قرمز **مختومه**  
- دلیل اجباری (مثلاً عدم علاقه‌مندی) + یک خط توضیح  
- از صف فعال خارج می‌شود ولی تاریخچه می‌ماند

## طراحی پنل — شیک، کاربردی، ریسپانسیو

### اصل‌ها
- فارسی، RTL، فونت خوانا (مثل Vazirmatn)
- موبایل‌اول: اپراتور اغلب با گوشی کار می‌کند
- حداکثر **۲ ضربه** تا ثبت تماس
- بدون کارت‌های تزئینی و آمار الکی
- رنگ فقط برای وضعیت؛ بقیه خنثی و تمیز
- دکمه‌های اصلی بزرگ و انگشتی (حداقل ۴۴px)

### چیدمان دسکتاپ (≥۱۰۲۴px)

```text
┌────────┬──────────────────────────────────────────────┐
│ سایدبار│  نوار بالا: جستجو ………………  نام اپراتور        │
│        │──────────────────────────────────────────────│
│ داشبورد│  [عقب‌افتاده ۵] [امروز ۱۲] [جدید ۸] [جلسه ۳] │
│ صف من  │                                              │
│ همه    │  لیست کارت‌مانند پرونده‌ها                    │
│ تقویم  │  نام · موبایل · مرحله · موعد · اکشن سریع     │
│ جستجو  │                                              │
└────────┴──────────────────────────────────────────────┘
```

### چیدمان موبایل (<۷۶۸px)

```text
┌─────────────────────────────┐
│ جستجو …………………              │
│ [عقب‌افتاده] [امروز] [جدید] │  ← چیپ افقی اسکرول
│                             │
│ کارت مشتری                  │
│ کارت مشتری                  │
│ …                           │
├─────────────────────────────┤
│ خانه │ صف │ تقویم │ جستجو │  ← نوار پایین ثابت
└─────────────────────────────┘
```

ثبت تماس روی موبایل = **شیت پایین (Bottom Sheet)** نه صفحهٔ جدا سنگین.

### صفحهٔ پرونده (قلب سیستم)

بالا:
- نام + تماس یک‌ضربه (`tel:`) + واتساپ
- نوار پیشرفت ۰–۱۰۰٪
- مرحله فعلی + مسئول + موعد شمسی

وسط — دو ستون دسکتاپ / یک ستون موبایل:
- **تایم‌لاین** (تماس، جلسه، تغییر مرحله)
- **اکشن‌ها**: ثبت تماس · ست جلسه · تغییر مرحله · مختومه

پایین:
- منبع لید (ارزیابی / رزرو / دستی)
- کشور و مقطع
- لینک به فرم ارزیابی (اگر باشد)

### صف‌ها (مرتب و خوانا)

تب‌ها:
1. همهٔ باز  
2. ارزیابی  
3. رزرو مشاوره  
4. ورود دستی  
5. مختومه

هر کارت:
`نام` · `۰۹۱۲…` · بج منبع · بج مرحله · موعد شمسی · آخرین نتیجه

اکشن سریع روی کارت (دسکتاپ هاور / موبایل سوایپ یا منوی ⋮):
- ثبت تماس  
- زنگ  
- موعد فردا  
- باز کردن پرونده

### تقویم شمسی

- نمای ماه / هفته / روز
- تاریخ جلالی روی هدر
- رویداد رنگی: اولیه · تخصصی · آنلاین · حضوری
- کلیک روی روز → لیست جلسات همان روز
- موبایل: هفته یا روز (ماه فقط خلاصه)

### جستجوی هوشمند

یک باکس بالا (دسکتاپ) یا تب پایین (موبایل):

پیدا می‌کند:
- نام (ناقص هم)
- موبایل با هر فرمت (`0912` / `912` / `+98`)
- کد پرونده
- کشور / مقطع
- تکهٔ یادداشت

پیشنهاد زنده زیر ۲ حرف؛ نتیجه با برجسته‌سازی تطبیق.

## اتصال به سیستم فعلی (بدون دور ریختن)

| موجود | استفاده در پنل |
|--------|----------------|
| `EvaluationRequest` + لاگ تماس | ورودی اصلی صف ارزیابی؛ همگام با پرونده |
| `ConsultationRequest` + اسلات | ساخت جلسه در تقویم شمسی |
| `ContactMessage` | لید ضعیف‌تر؛ قابل تبدیل به پرونده |

لایهٔ جدید:
`Customer` → `CustomerCase` → `CaseEvent` + `CaseAppointment`

قانون طلایی: **یک شماره موبایل = یک مشتری** (تکراری‌ها ادغام می‌شوند).

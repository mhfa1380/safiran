# راهنمای سئو سایت سفیران آینده روشن

## تغییرات اعمال‌شده

### ۱. آدرس‌های سئو‌فرندلی (URL با کلمات کلیدی فارسی)
- `/درباره-ما/` — درباره موسسه
- `/تماس-با-ما/` — تماس با ما
- `/دانشگاه-های-خارج/` — لیست دانشگاه‌ها
- `/دانشگاه/<slug>/` — جزئیات دانشگاه
- `/رشته-های-تحصیلی/` — لیست رشته‌ها
- `/رشته/<slug>/` — جزئیات رشته
- `/دوره-های-تحصیلی/` — لیست دوره‌ها
- `/دوره/<slug>/` — جزئیات دوره
- `/رزرو-مشاوره/` — رزرو وقت مشاوره
- `/ارزیابی-مهاجرت/` — فرم ارزیابی
- `/سوالات-متداول/` — سوالات متداول
- `/خدمات-موسسه/` — خدمات موسسه

آدرس‌های قدیمی (مثل `/about/`, `/contact/`) با ریدایرکت ۳۰۱ به آدرس جدید هدایت می‌شوند.

### ۲. Sitemap و Robots
- **`/sitemap.xml`**: نقشه سایت شامل صفحات استاتیک، وبلاگ، دانشگاه‌ها، دوره‌ها و رشته‌ها
- **`/robots.txt`**: راهنمای خزنده‌ها با لینک به sitemap و مسیرهای غیرمجاز (admin، ckeditor)

### ۳. FAQ Schema
- صفحه سوالات متداول دارای داده ساختاریافته `FAQPage` برای نمایش در نتایج گوگل (Rich Snippets)

### ۴. متاتگ‌های سئو (در `base.html`)
- **Canonical URL**: جلوگیری از محتوای تکراری
- **Meta robots**: `index, follow` برای ایندکس شدن
- **Open Graph**: برای اشتراک‌گذاری در شبکه‌های اجتماعی
- **Twitter Card**: کارت بزرگ با تصویر برای توییتر

### ۵. داده ساختاریافته (JSON-LD)
- **Organization**: اطلاعات موسسه
- **WebSite**: اطلاعات سایت
- **CollegeOrUniversity**: برای صفحات دانشگاه‌ها
- **BlogPosting**: برای پست‌های وبلاگ
- **Course**: برای دوره‌ها
- **EducationalOccupationalProgram**: برای رشته‌ها
- **BreadcrumbList**: مسیر ناوبری برای صفحات جزئیات

### ۶. سلسله‌مراتب هدینگ و تصاویر
- هر صفحه یک `h1` منحصر به فرد
- تمام تصاویر دارای `alt` توصیفی
- `loading="lazy"` برای تصاویر زیر تاشو

### ۷. صفحات بهینه‌شده
- صفحه اصلی، درباره ما، تماس، وبلاگ، دانشگاه‌ها، رشته‌ها، دوره‌ها، خدمات، رزرو، ارزیابی، FAQ
- هر صفحه دارای meta description و title منحصر به فرد

## کارهای لازم قبل از انتشار

### ۱. تنظیم SITE_URL
در `settings.py` مقدار `SITE_URL` با آدرس واقعی سایت تنظیم شده است:

```python
SITE_URL = "https://www.saroshan.ir"
```

**مهم:** این آدرس برای canonical، og:url و JSON-LD استفاده می‌شود و حتی در تست لوکال هم باید آدرس واقعی باشد (نه localhost).

### ۲. به‌روزرسانی Site
در پنل ادمین Django → Sites → دامنه را با آدرس واقعی تنظیم کنید.

### ۳. تصویر پیش‌فرض OG
- برای اشتراک‌گذاری در شبکه‌های اجتماعی، تصویری با ابعاد `1200×630` پیکسل در مسیر `static/img/og-default.png` قرار دهید.
- در صورت نبود، از لوگو استفاده می‌شود.

### ۴. Google Search Console
- سایت را در [Google Search Console](https://search.google.com/search-console) ثبت کنید.
- فایل `sitemap.xml` را ارسال کنید.

## نکات مهم برای رتبه‌بندی

1. **محتوای باکیفیت**: محتوای منسجم و مفید برای کاربران بنویسید.
2. **کلمات کلیدی**: از کلمات کلیدی مرتبط (اعزام دانشجو، مهاجرت تحصیلی، کانادا، چین، اسپانیا) در عنوان و توضیحات استفاده کنید.
3. **سرعت سایت**: سرعت بارگذاری را با بهینه‌سازی تصاویر و کش بهبود دهید.
4. **لینک‌های داخلی**: بین صفحات مرتبط لینک دهید.
5. **به‌روزرسانی منظم**: محتوای وبلاگ و صفحات را به‌طور منظم به‌روز کنید.
